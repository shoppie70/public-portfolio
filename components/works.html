Instagram Graph API Token Generator

🔗 サイトにアクセスする →

インスタグラムの投稿をWebページに埋め込む際、Instagram Graph APIを利用してトークンを発行する必要があります。

頭痛管理Webアプリ「ずきメモ」

「痛すぎて滅、治したくて記録。」

🔗 サイトにアクセスする →

既存の頭痛記録アプリに存在する「服薬ログの紐付けバグ」や「データエクスポートの有料制限」といった実用上のペインを根本から解消するために開発した、モダンモノリス構成のモバイルファーストWebアプリケーション（将来的なReact Nativeへの移行を視野に入れた設計）。

課題解決・UXへのこだわり:
  • バグゼロの追求とUXの両立: 頭痛発生時の認知負荷を下げるため、1〜2タップで『同時に服用した薬・量・時刻』を直感的に記録できるワンモーダルUIを構築。
  • 実用的な共有機能: ファミリーリンク（パートナー共有）機能を標準搭載。夫婦間での体調可視化・共有を自動化し、シフト制での乳児の育児・お世話における連携プレイをシステム面から強固にサポート。
技術的工夫:
  • Inertia.jsによるモダンモノリス: APIのエンドポイント開発や二重ルーティングを完全に排し、LaravelコントローラーからReactコンポーネントへ直接Propsでデータを流し込むSPA構成を採用。開発スピードとシームレスなUXを極限まで両立。
  • 厳格なDBリレーション設計: headache_logs（頭痛）とmedication_logs（服薬）をDBレベルで明確に分離。1対多のリレーションを定義し、予防薬の単独服薬にも対応しつつ、紐付け表示バグを構造的に排除。
  • 外部API連携によるデータ自動補完: フロントのGeolocation APIで取得した緯度経度ベースに、気象情報APIへ非同期リクエスト。その時点の天気・気圧・気温を自動補完し、PDFレポート出力や気圧相関グラフを完全無料で出力可能。

潮見表ツール「シオヨミ」

🔗 サイトにアクセスする →

自身がよく行く兵庫・鳥取・島根・岡山・香川などの主要な港（20箇所）の『潮汐・天気・風速』を、複数サイトを行き来することなく1画面で瞬時に把握するために開発した釣行計画立案ツール。

課題解決・UXへのこだわり:
  • 釣果に直結するデータの統合可視化: 24時間の潮位グラフ（Chart.js）と、風向き・風速（m/s）データを同一グラフ上に重畳表示。月間カレンダービューとも連動させ、最小限の画面遷移で釣行判断に必要な情報を集約。
技術的工夫:
  • プレーンPHPによるレイヤードアーキテクチャ: あえてフルスタックフレームワークに依存せず、PHP 8.0単体で構築。ディレクトリ構造を Requests / Services / UseCases に明示的に分離し、疎結合で保守性の高いオブジェクト指向設計を自力で具現化。
  • マルチAPIオーケストレーション: tide736.net から潮汐データを、Open-Meteo から気象・風速データをバックエンドで並行取得し、高速にパース・統合する軽量データパイプラインを実装。
  • ポータブルな開発環境: Apache + PHP 8.0のDockerコンテナ環境を内製し、ローカルから本番（Xserver）へのデプロイまで環境差分のない堅牢な運用を担保。

YouTube「高嶺のしょぴこさん♪」運営

🔗 チャンネルにアクセスする →

感覚や運に頼る動画投稿ではなく、エンジニア的な定量的データ分析をYouTubeアルゴリズムに適用し、戦略的にグロースさせた釣魚特化型YouTubeメディア。

課題解決・UXへのこだわり:
  • 緻密なデータドリブンPDCA: インプレッションクリック率（CTR）、視聴維持率（リテンション）、トラフィックソース等のアナリティクスデータを日々トラッキングし、高速で仮説検証サイクル（PDCA）を回す仕組みを自走。
技術的工夫:
  • ニッチ市場 of セグメンテーション: 『釣り』という巨大なレッドオーシャンの中で、地域や特定の釣法に特化したセグメントを定義。競合優位性の高いコンテンツポジションを戦略的に確立。
  • CTR/リテンションの最大化施策: クリック率向上のためのサムネイル（構図・配色）の比較検証、および視聴維持率を最大化するための動画構成のデータ最適化を徹底。結果として、チャンネル開設からわずか約半年でチャンネル登録者数1,000人を突破し、収益化要件を最速でクリア。ゼロからのメディア立ち上げ力とプロダクトグロース能力を実証。

広島市保活支援検索ツール「ほいサーチ」

「自治体の複雑な公開情報を、ユーザー目線で徹底的に構造化」

🔗 サイトにアクセスする →

広島市の自治体公式データを活用し、スマートフォンでの閲覧ストレス解消と、保護者が本当に必要とする条件による高速絞り込みに特化させた保活支援Webアプリケーション。

課題解決・UXへのこだわり:
  • モバイル最適化と視認性改善: 自治体公式の複雑な一覧表や大容量PDFが持つ「スマートフォンからでは著しく読みづらい」というペインを、レスポンシブなUI設計により根本から解消。
  • 保護者ファーストのこだわり絞り込み: 「おむつの園内処分可能か」「布団の持ち込み不要か」「駐車場の有無」など、従来の公開情報では埋もれがちだった保護者のコアな判断基準を1タップでサクサクフィルタリングできる検索ロジックを実装。
  • 直感的な基本・延長保育時間の可視化: 各施設の預かり時間をグラフィカルなタイムライン形式に落とし込み、複数施設の比較検討を直感的に行えるUXを追求。
技術的工夫・データの透明性:
  • 公式ソースとの透過的連携: 非公式ツールでありながら情報の正確性と信頼性を担保するため、元データである広島市公式のデータソースURLへダイレクトに遷移・確認できる透過的なリンク設計。
  • エッジホスティングによる超高速配信: インフラにCloudflare Pagesを採用。地域ごとに最適化されたデータを高速配信し、子育て中の慌ただしい時間でも一切の待ち時間を感じさせない高速なページ表示速度を実現。
